Password Protection

Introduction

A password is a series of letters, numbers and symbols used to authenticate a user when logging into to a system. Passwords are used alongside a username when logging in and are designed to only be known by the user who is authenticating with the system.

Why are passwords important?

Would you leave your front door or window open on your house? That is essentially what you are doing when you create a weak password for your user accounts. One of the most common ways hackers access personal accounts is through guessing weak passwords.

Common password attacks

Below are the common ways passwords are hacked:

  • Dictionary attacks - A brute force attack to gain unauthorised access to a system by trying hundreds of possible password combinations
  • Social engineering - The art of manipulating people so they give up confidental information such as passwords
  • Guessing simple passwords - Hackers can access to systems by using common passwords such as 123456, qwerty and password
  • Reusing passwords - When a hacker gains access to one system, that same password can be used to hack into other accounts

Password best practises

Follow the below best practises when creating passwords:

  • Passwords should be at least 8 characters long
  • They should contain a combination of letters (both uppercase and lowercase), numbers and special characters
  • A password should never be written down or shared with anyone else
  • Never use the same password for multiple accounts
  • Passwords should not include personal information such as name, birthday or other easily guessable data
